# AI 讨论室规划索引

本文档集用于指导从零实现一个多人在线 AI 讨论室。系统目标是让用户创建讨论室，邀请多个具备不同分工的 AI 角色进入房间，围绕议题协作讨论，并在后续支持狼人杀等规则游戏。

## 已确认决策

- 产品形态：多人在线 Web 服务。
- AI 接入：同时支持 mock 与真实 Provider；实现顺序为先 mock，后真实 API。
- 第一阶段：只完成规划文档，不实现业务代码。
- 实现原则：代码命名简单准确，非必要不新增函数、参数、注释或抽象。

## 阶段目标

| 阶段 | 目标 | 产物 |
| --- | --- | --- |
| 规划 | 明确边界、架构和实施顺序 | 本文档集 |
| 骨架 | 建立可运行 Web 项目 | 首页和基础工程配置 |
| 领域 | 跑通最小房间模型 | 内存版房间、成员、消息 |
| Mock 讨论 | 验证多 AI 讨论体验 | mock AI 轮流发言 |
| 实时 | 支持多人在线同步 | WebSocket 房间事件 |
| 持久化 | 支持刷新恢复和历史记录 | 数据库表和查询接口 |
| Provider | 接入真实模型 | 统一 Provider 与预算控制 |
| 游戏 | 支持狼人杀等规则玩法 | 通用规则引擎和具体游戏 |

## 总体原则

- 先做讨论室，再做游戏。
- 先做顺序发言，再做并行发言。
- 先用 mock 验证流程，再接真实模型。
- 先做单体 Web 服务，再按实际压力拆分。
- 先记录事件日志，不一开始实现完整事件溯源。
